## Структура задания

### Часть 1. Реализация

Разработайте рабочее решение для **одной** задачи из трёх:

| Задача | Пример компонента (либо указываем свой) |
|--------|-------------------------------------------|
| **Анализ научных статей** (кластеризация) | KMeans / Gaussian Mixture / DBSCAN |
| **Маршрутизация запросов** (классификация) | Logistic Regression / Random Forest / SVM |
| **Рубрикация комментариев** (классификация) | Random Forest / Logistic Regression |

**Требования:**

1. Использовать **Yandex API** (YandexGPT) для бейзлайна.
2. Реализовать два подхода в одном ноутбуке/скрипте:
   - `baseline`: LLM решает задачу «в лоб» (без вашего ML-компонента)
   - `proposed`: ваше решение с традиционным ML
3. Сравнить их **качественно** (на 5–10 примерах).
4. Для замера скорости работы baseline используйте логирование времени выполнения запросов к Yandex API (например, `time.time()` до и после вызова). Стоимость оцените в токенах (YandexGPT возвращает количество использованных токенов в ответе).

### Часть 3. Вывод ответов модели

Выведите для 5–10 примеров сравнение в читаемом виде.  
Можно использовать `print()`, Markdown-таблицы в Colab, HTML-виджеты — на ваше усмотрение.

**Пример кода для вывода (через `print`):**

```python
def print_comparison(examples, baseline_preds, proposed_preds):
    print(f"{'Пример':<50} | {'LLM (baseline)':<20} | {'ML (proposed)':<20}")
    print("-" * 100)
    for ex, base, prop in zip(examples, baseline_preds, proposed_preds):
        print(f"{ex[:47]:<50} | {base:<20} | {prop:<20}")
```
